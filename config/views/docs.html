API Documentation

Everything required to authenticate and call LibiWeb API v2 endpoints.

Base URL

https://api.libiweb.com

Authentication

All /v2/* routes require the x-api-key request header.

x-api-key: your-api-key

Endpoints

  • GET /api - public service metadata.
  • GET /swagger.json - OpenAPI specification (public).
  • GET /v2/health - health check (requires API key).
  • GET /v2/status - runtime status (requires API key).
  • POST /v2/crawl - crawl target URL via Crawl4AI (requires API key).
  • POST /map - mapped URLs grouped into internal/external from Crawl4AI link map output (requires API key).
  • POST /metadata - normalized metadata records from Crawl4AI output (requires API key).
  • POST /getmedia - normalized media assets (images/videos/audios) from Crawl4AI output (requires API key).
  • POST /v2/metadata - versioned alias for normalized metadata records (requires API key).
  • POST /v2/getmedia - versioned alias for normalized media assets (requires API key).

Example

curl -H "x-api-key: your-api-key" https://api.libiweb.com/v2/health

Crawl Example

curl -X POST "https://api.libiweb.com/v2/crawl" \
  -H "Content-Type: application/json" \
  -H "x-api-key: your-api-key" \
  -d '{"urls":["https://example.com"],"formats":["markdown"]}'
