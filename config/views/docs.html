API Documentation

Everything required to authenticate and call LibiWeb API v2 endpoints.

Base URL

https://api.libiweb.com

Authentication

All /v2/* routes require the x-api-key request header.

x-api-key: your-api-key

Endpoints

  • GET /api - public service metadata.
  • GET /swagger.json - OpenAPI specification (public).
  • GET /v2/health - health check (requires API key).
  • GET /v2/status - runtime status (requires API key).
  • POST /v2/crawl - crawl and extraction response for target URLs (requires API key).
  • POST /v2/map - internal/external link mapping (requires API key).
  • POST /v2/metadata - normalized metadata records (requires API key).
  • POST /v2/getmedia - normalized media assets (requires API key).
  • POST /v2/getmarkdown - normalized markdown output (requires API key).
  • POST /v2/gethtml - normalized HTML output (requires API key).
  • POST /v2/gettext - plain text extraction (requires API key).
  • POST /v2/getseo - SEO-focused extraction (requires API key).
  • POST /v2/getemails - email discovery from target and linked pages (requires API key).
  • POST /v2/extract - built-in field extraction (requires API key).
  • POST /v2/screenshot - screenshot extraction (requires API key).
  • POST /v2/bulk - queue bulk crawl job (requires API key).
  • GET /v2/bulk/:jobId - bulk job status/result (requires API key).
  • GET /v2/usage - usage analytics summary (requires API key).
  • POST /v2/webhook/register - register bulk webhook callback (requires API key).

Example

curl -H "x-api-key: your-api-key" https://api.libiweb.com/v2/health

Crawl Example

curl -X POST "https://api.libiweb.com/v2/crawl" \
  -H "Content-Type: application/json" \
  -H "x-api-key: your-api-key" \
  -d '{"urls":["https://example.com"],"formats":["markdown"]}'
